# AGENTS.md — Helm Charts Repository

This repository contains Helm charts for Kubernetes deployments. The primary charts are
`buildkit-service` and `smocker`, located under `charts/`.

---

## Toolchain

All tasks are managed via **[mise](https://mise.jdx.dev/)** (no Makefile or npm scripts).
Tool versions are pinned in `mise.toml`:

| Tool         | Purpose                                  |
|--------------|------------------------------------------|
| helm         | Chart rendering, linting, dependency mgmt |
| helm-docs    | Auto-generate README from values.yaml    |
| kubeconform  | Validate rendered templates against k8s schema |
| pre-commit   | Git hooks: helm-docs + helm-lint         |

Install all tools and pre-commit hooks:

```bash
mise install
```

## Build / Lint / Validate Commands

### Lint a chart (all CI values combinations)

```bash
ct lint --config .github/config/ct-lint.yaml --check-version-increment=false
```

### Lint a single chart manually with a specific values file

```bash
helm lint -f charts/buildkit-service/ci/ci-default-values.yaml charts/buildkit-service
helm lint -f charts/smocker/ci/ci-default-values.yaml charts/smocker
```

### Validate rendered templates against Kubernetes schemas (kubeconform)

```bash
# Pass version as a CLI flag
mise kubeconform --kubernetes-version <version>

# Validate a specific chart (skips changed-charts detection)
mise kubeconform --kubernetes-version <version> --chart buildkit-service

# Version can also be passed via env (e.g. in CI)
KUBERNETES_VERSION=<version> mise kubeconform
```

Supported test matrix versions are defined in `.github/workflows/main.yml` under the `kubernetes_version` matrix.

### Validate a single chart + values file manually

```bash
helm dependency update charts/buildkit-service
helm template charts/buildkit-service -f charts/buildkit-service/ci/ci-default-values.yaml \
  | kubeconform -strict -verbose -kubernetes-version <version>
```

### Lint docs (ensure README.md is up to date)

```bash
mise lint-docs
```

### Regenerate docs

```bash
helm-docs -c charts
```

### Bump chart version (patch/minor/major)

```bash
mise bump-version --chart buildkit-service --version-segment patch
```

## CI Pipeline (`.github/workflows/main.yml`)

Triggered on PRs and pushes to `main` touching `charts/**`:

1. **lint-docs** — runs `mise lint-docs`; fails if README is out of date
2. **lint** — runs `ct lint` (chart-testing)
3. **validate** — runs `mise kubeconform` for each k8s version in the matrix
4. **install** — creates a `kind` cluster and runs `ct install` (only when charts change)

A separate `update-docs` job auto-commits regenerated docs if `lint-docs` fails on a PR.

## CI Test Fixtures

Each chart has CI values files under `charts/<name>/ci/ci-*.yaml`. These act as test cases —
every lint, kubeconform validation, and install run iterates over all matching `ci-*.yaml` files.

## Pre-commit Hooks

The `.pre-commit-config.yaml` runs two hooks automatically on `git commit`:

1. **helm-docs** — regenerates `README.md` for any changed chart
2. **helm-lint** (`hooks/helmlint.sh`) — runs `helm lint` for each `ci-*.yaml` combination
   against any changed chart file

To run hooks manually:

```bash
pre-commit run --all-files
```

## Helm Chart Structure

Each chart follows standard Helm layout:

```
charts/<name>/
├── Chart.yaml            # Chart metadata: name, version, appVersion, description
├── values.yaml           # Default values; all keys documented with `# --` comments
├── README.md             # Auto-generated by helm-docs from README.md.gotmpl
├── README.md.gotmpl      # helm-docs template using chart.* template functions
├── templates/
│   ├── _helpers.tpl      # Named templates (helpers): name, fullname, labels, etc.
│   ├── deployment.yaml   # Main workload template
│   └── ...               # Other Kubernetes resource templates
└── ci/
    └── ci-*.yaml         # CI test values files
```

## Code Style Guidelines

### Helm Templates (Go templates)

**Named templates (`_helpers.tpl`)**

- Namespace all template names with the chart shortname: `"buildkit.name"`, `"smocker.labels"`
- Use `trunc 63 | trimSuffix "-"` on all name/fullname helpers (DNS label limit)
- Use `required` for any value that must be provided by the user:

  ```
  {{- required "Certificate is required when TLS is enabled" .Values.tls.cert -}}
  ```

- Use `semverCompare` for Kubernetes version-gated API differences

**Template files**

- Use `{{- if }}` / `{{- with }}` / `{{- range }}` with leading dash to trim whitespace
- Use `nindent` (not `indent`) for all multi-line YAML blocks to include the leading newline:

  ```yaml
  labels:
    {{- include "buildkit.labels" . | nindent 4 }}
  ```

- Use `toYaml | nindent` for passthrough values (resources, nodeSelector, tolerations, affinity):

  ```yaml
  resources:
    {{- toYaml .Values.resources | nindent 12 }}
  ```

- Wrap optional sections with `{{- with .Values.someKey }}` to suppress empty output
- Conditionalize features with boolean values in `values.yaml` (e.g., `tls.enabled`, `autoscaling.enabled`)
- Use inline conditionals (`ternary`) sparingly and only for simple string switches

**Standard labels block** — always include both `labels` and `selectorLabels` helpers:

```yaml
labels:
  {{- include "buildkit.labels" . | nindent 4 }}
```

```yaml
matchLabels:
  {{- include "buildkit.selectorLabels" . | nindent 6 }}
```

### `values.yaml`

- Document every key with a `# --` comment immediately above the key (helm-docs convention):

  ```yaml
  # -- Enable mTLS
  enabled: false
  ```

- Group related values under a nested map (e.g., `tls:`, `autoscaling:`, `buildkitVolume:`)
- Provide commented-out examples for list/map values:

  ```yaml
  env: []
    # - name: HTTP_PROXY
    #   value: "http://proxy.example.com:8080"
  ```

- Use empty string `""` as default for optional string values, not `null`

### `Chart.yaml`

- Maintain `annotations.artifacthub.io/links` with source and support URLs
- `version` is the chart semver (bumped via `mise bump-version`)
- `appVersion` is the upstream app version (updated by Renovate bot)

### `README.md.gotmpl`

- Use helm-docs template functions: `chart.header`, `chart.versionBadge`, `chart.valuesSection`, etc.
- Keep human-written intro/installation sections; let helm-docs generate the values table

### Bash Scripts (`.mise-tasks/`, `hooks/`)

- Shebang: `#!/usr/bin/env bash` (tasks) or `#!/bin/bash` (hooks)
- Always `set -e` as first statement after shebang/comments
- Source shared utilities: `source "${MISE_PROJECT_ROOT}/.mise-tasks/utils.sh"`
- Use `local -r` for immutable local variables in functions
- Use `[[ ]]` double-bracket conditionals (not `[ ]`)
- Log output via the `log()` helper (yellow ANSI color), not raw `echo`
- Add `#MISE description="..."` annotation for task discoverability

## Versioning & Release

- Chart versions follow semver; bump via `mise bump-version --chart <name> --version-segment <patch|minor|major>`
- `appVersion` in `Chart.yaml` tracks the upstream app version; updated automatically by Renovate
- Release is triggered by the `helm chart-releaser` action on merge to `main` after a version bump commit

---

## Dependency Updates

- **Renovate** manages `appVersion` for `buildkit-service`, and pinned versions of `kubeconform`
  and `helm-docs` in `utils.sh` via custom regex managers
- **Dependabot** manages GitHub Actions and Docker image versions
